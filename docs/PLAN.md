# PLAN — Phase별 개발 계획 (초안)

`docs/PRD.md`, `docs/FEATURES/main.md`, `docs/FEATURES/game_rule.md`, `docs/FEATURES/mission1.md`에서 정한 내용을 바탕으로, Phase를 잘게 나눠 개발한다.

각 Phase는 **직접 실행해서 확인할 수 있는 동작되는 SW**를 목표로 한다. Phase가 끝날 때마다 고객님(사용자)이 직접 실행해보고, 아래 "고객님이 확인할 부분"을 중심으로 테스트 후 피드백을 준다. 피드백 결과에 따라 다음 Phase 진행 전 이 문서(PLAN.md)의 내용을 수정할 수 있다.

## Phase 1: 메인 화면 UI (정적 화면)

- **목표 기능**: 앱 실행 시 메인 화면만 뜨는 상태 (`docs/FEATURES/main.md` 기준)
  - 타이틀(PANG) 표시
  - 게임 선택 영역 표시 (현재는 Mission 1 하나만 존재)
  - 시작 버튼 표시
  - 종료 버튼 표시
- **고객님이 확인할 부분**
  - 앱 실행 시 메인 화면이 정상적으로 뜨는지
  - 타이틀/게임 선택/시작 버튼/종료 버튼이 화면에 모두 보이는지, 레이아웃이 어색하지 않은지
  - (이 단계에서는 버튼을 눌러도 아직 동작하지 않아도 된다는 점 확인)

## Phase 2: 메인 화면 버튼 동작 연결

- **목표 기능**: Phase 1에서 만든 버튼들이 실제로 동작함
  - 게임 선택 영역에서 Mission 1을 선택할 수 있음
  - 시작 버튼을 누르면 (아직 내용은 비어있는) 게임 화면으로 전환됨
  - 종료 버튼을 누르면 게임 종료 동작이 실행됨
- **고객님이 확인할 부분**
  - 게임 선택이 정상적으로 되는지 (선택된 상태가 화면에 표시되는지)
  - 시작 버튼 클릭 시 게임 화면으로 전환되는지
  - 종료 버튼 클릭 시 의도한 종료 동작이 실행되는지

## Phase 3: Player 이동

- **목표 기능**: 게임 화면에 Player가 보이고 좌우로 이동
  - Player가 방향키 좌/우 입력에 따라 화면 안에서 이동
  - 화면 경계를 벗어나지 않음
- **고객님이 확인할 부분**
  - 방향키 입력에 맞춰 Player가 자연스럽게 이동하는지
  - 화면 좌/우 끝에서 더 이상 밀려나가지 않고 멈추는지

## Phase 4: Wire 발사

- **목표 기능**: Player가 Wire를 발사할 수 있음
  - Space 키로 Player 위치에서 수직 위로 Wire 발사
  - Wire가 화면 상단에 닿으면 사라짐 (Bubble/Block 아직 없음)
- **고객님이 확인할 부분**
  - Space 키 입력 시 Wire가 정확히 수직으로 발사되는지
  - Wire가 화면 상단에서 자연스럽게 사라지는지
  - 연속 발사 시 딜레이나 끊김이 어색하지 않은지

## Phase 5: Bubble 기본 물리

- **목표 기능**: Bubble이 화면에 등장해 튕기며 움직임 (Block/충돌 처리 없이 관찰용)
  - Bubble이 중력의 영향을 받아 포물선으로 튕김
  - 화면 좌/우/상단 경계에서 반사되고 밖으로 나가지 않음
- **고객님이 확인할 부분**
  - Bubble이 튕기는 움직임(높이, 속도)이 자연스러운지
  - 화면 경계를 벗어나거나 화면 밖으로 사라지지 않는지

## Phase 6: Block 배치와 충돌

- **목표 기능**: Block이 배치되고 Bubble/Wire와 충돌함
  - Block이 파괴되지 않는 고정 장애물로 화면에 배치됨
  - Bubble이 Block 위쪽 면에서 튕겨 올라감
  - Wire가 Block에 닿으면 그 지점에서 사라짐 (Block을 뚫지 못함)
- **고객님이 확인할 부분**
  - Block의 위치가 시각적으로 장애물로 명확히 인식되는지
  - Bubble이 Block을 통과하지 않고 튕기는지
  - Wire가 Block 앞에서 정확히 멈추는지

## Phase 7: Wire와 Bubble 충돌 (분할 규칙)

- **목표 기능**: Wire가 Bubble을 맞추면 규칙대로 분할/소멸함 (`docs/FEATURES/game_rule.md` 기준)
  - 일반 크기 Bubble이 Wire에 맞으면 더 작은 Bubble 2개로 분할
  - 가장 작은 크기 Bubble이 Wire에 맞으면 소멸
- **고객님이 확인할 부분**
  - Wire와 Bubble의 충돌 판정이 눈에 보이는 위치와 일치하는지
  - 분할된 Bubble의 크기와 개수가 규칙대로 나오는지
  - 가장 작은 Bubble이 맞았을 때 정상적으로 사라지는지

## Phase 8: Player와 Bubble 충돌, 목숨 관리

- **목표 기능**: Player가 Bubble에 닿으면 목숨이 줄고, 목숨 소진 시 게임 오버 처리
  - Player와 Bubble 접촉 판정
  - 목숨 감소 표시
  - 목숨이 모두 소진되면 게임 오버 화면으로 전환
- **고객님이 확인할 부분**
  - Player가 Bubble에 닿는 순간이 정확히 판정되는지
  - 목숨 감소가 화면에 바로 반영되는지
  - 목숨이 0이 됐을 때 게임 오버 화면으로 넘어가는지

## Phase 9: 승리 조건과 결과 화면

- **목표 기능**: 모든 Bubble을 제거하면 승리 처리
  - 화면의 모든 Bubble(분할된 것 포함) 제거 시 승리 판정
  - 승리/게임 오버 화면에서 재시작 또는 메인 화면으로 이동 가능
- **고객님이 확인할 부분**
  - 마지막 Bubble을 제거한 시점에 바로 승리 처리가 되는지
  - 승리 화면과 게임 오버 화면이 각각 구분되어 보이는지
  - 결과 화면에서 재시작/메인으로 이동하는 동작이 정상적으로 되는지

## Phase 10: Mission 1 난이도 적용 및 전체 흐름 연결

- **목표 기능**: Mission 1 난이도를 반영하고 전체 흐름을 하나로 연결 (`docs/FEATURES/mission1.md` 기준)
  - Mission 1의 초기 Bubble 개수/크기, Block 배치 적용
  - 메인 화면 → 게임 선택 → 시작 → Mission 1 플레이 → 승리/게임 오버 → 메인 화면 복귀까지 끊김 없이 연결
- **고객님이 확인할 부분**
  - 메인 화면부터 결과 화면까지 전체 흐름이 끊기지 않고 자연스러운지
  - Mission 1의 난이도가 "처음 배우기 좋은 쉬운 난이도"로 느껴지는지
  - 종료 버튼을 포함한 전체 UI 동작이 처음부터 끝까지 문제없이 이어지는지

## Phase 11: 게임 선택 목록에 Mission 2~5 추가

- **목표 기능**: 메인 화면의 "게임 선택" 목록에 Mission 2, 3, 4, 5를 추가로 노출
  - 방향키로 Mission 1~5까지 모두 포커스 이동 가능
  - 아직 난이도 데이터가 없는 Mission 2~5는 임시로 Mission 1과 동일한 난이도로 플레이됨(다음 Phase에서 순서대로 실제 난이도로 교체)
- **고객님이 확인할 부분**
  - 게임 선택 목록에 Mission 1~5가 모두 보이는지, 방향키로 전부 이동 가능한지
  - Mission 2~5 중 아무거나 선택해서 시작해도 게임 화면으로 정상 진입하는지 (난이도는 임시로 Mission 1과 같아도 됨)

## Phase 12: Mission 2 난이도 적용

- **목표 기능**: Mission 2를 선택하면 Mission 1보다 살짝 어려운 실제 난이도로 플레이됨
  - 큰 Bubble 2개로 시작 (서로 반대 방향으로 튕겨나감), Block 없음
- **고객님이 확인할 부분**
  - Mission 2 시작 시 큰 Bubble이 2개 등장하는지
  - Mission 1보다 난이도가 자연스럽게 올라간 느낌인지(너무 급격하지 않은지)

## Phase 13: Mission 3 난이도 적용

- **목표 기능**: Mission 3에 Block(장애물)이 처음 등장
  - 큰 Bubble 2개 + Block 1개 배치
- **고객님이 확인할 부분**
  - Block이 화면 중간 높이에 배치되어 Bubble이 튕기는 경로에 실제로 영향을 주는지
  - Block이 Wire 사거리를 가리는 위치에 있어 전략적으로 피하거나 이용해야 하는지

## Phase 14: Mission 4 난이도 적용

- **목표 기능**: Bubble 개수/크기 조합과 Block 배치가 더 복잡해짐
  - 큰 Bubble 2개 + 중간 Bubble 1개, Block 2개(서로 다른 높이)
  - Bubble 수평 이동 속도를 Mission 1~3보다 약간 높임
- **고객님이 확인할 부분**
  - Bubble 종류(큰 것/중간 것)가 섞여 있어도 분할 규칙이 각각 올바르게 동작하는지
  - Block 2개가 배치된 상태에서도 플레이가 막히지 않고 클리어 가능한지

## Phase 15: Mission 5 난이도 적용 (최종 난이도)

- **목표 기능**: 지금까지 중 가장 어려운 구성 (KATA 범위 내 최종 난이도)
  - 큰 Bubble 2개 + 중간 Bubble 2개, Block 3개로 좀 더 복잡한 배치
  - Bubble 수평 이동 속도를 Mission 4보다 한 단계 더 높임
- **고객님이 확인할 부분**
  - Mission 1→5로 갈수록 난이도가 순차적으로 올라가는 느낌이 드는지
  - Mission 5가 어렵지만 클리어 불가능할 정도는 아닌지(막히는 구간이 없는지)

## 참고 / 가정

- Phase 구분과 각 Phase의 세부 범위는 초안이며, 고객님 피드백에 따라 조정될 수 있다.
- "게임 선택" 영역은 현재 Mission 1 하나만 존재한다고 가정하고, 추후 미션이 추가되면 목록이 늘어나는 구조로 설계한다.
- Mission 2~5의 구체적인 Bubble/Block 배치 수치는 "단계적으로 조금씩 어려워진다"는 방향성 아래 임시로 정한 값이며, 실제 플레이 후 조정될 수 있다.
