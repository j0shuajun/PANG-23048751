# Phase 16 설계 — 미션 선택 좌우 방향키 전환 (초안)

`docs/PLAN.md`의 Phase 16 범위를 어떻게 구현할지 정리한 설계 문서다. Mission이 5개로 늘어나면서, 기존처럼 미션 항목 5개를 시작/종료 버튼과 함께 위/아래로 모두 나열하는 방식은 이동이 번거로워졌다. 이를 개선한다.

## 1. 범위

- **포함**: 메인 화면의 키보드 메뉴 구조를 3개 행(게임 선택 / 시작 / 종료)으로 재구성하고, 게임 선택 행에서는 좌/우 방향키로 Mission을 전환하도록 변경
- **제외**: 시작/종료 버튼이 실행되는 방식 자체(Phase 2에서 만든 로직)는 그대로 유지

## 2. 기존 방식과의 차이

| 구분 | 기존 (Phase 1~2) | 변경 후 (Phase 16) |
| --- | --- | --- |
| 위/아래 방향키 이동 대상 | Mission 1~5 + 시작 + 종료, 총 7개 항목 | 게임 선택 / 시작 / 종료, 총 3개 행 |
| 미션 전환 방법 | 위/아래로 미션 항목까지 이동 후 Enter로 선택 | 게임 선택 행에 포커스된 상태에서 좌/우로 즉시 전환 |
| 현재 선택된 미션 표시 | 목록에서 굵은 글씨로 표시 | 가운데 미션 이름 하나만 표시(`◀ Mission N ▶`) |

## 3. 모듈 구조

```
src/screens/MainScreen.tsx  (변경) 메뉴 구조를 3행으로 재구성, 좌/우 키 처리 추가
src/screens/MainScreen.css  (변경) 게임 선택 영역을 캐러셀 형태로 스타일링
```

## 4. 상세 설계

- 포커스 대상은 `focusedRow` state(0: 게임 선택, 1: 시작, 2: 종료)로 관리한다. 항목 수가 3개로 고정되므로 기존 `focusedIndex` + `menuItems` 배열 방식 대신 단순한 3행 모델로 바꾼다.
- 현재 선택된 미션은 `selectedMissionIndex`(0 ~ `missions.length - 1`) state로 관리한다.
- 키 입력 처리:
  - `ArrowDown` / `ArrowUp`: `focusedRow`를 3을 기준으로 순환 이동(기존과 동일한 순환 로직, 대상 개수만 3개로 축소)
  - `ArrowRight` / `ArrowLeft`: `focusedRow === 0`(게임 선택 행에 포커스)일 때만 `selectedMissionIndex`를 `missions.length` 기준으로 순환 이동. 다른 행에 포커스가 있을 때는 좌/우 입력을 무시한다.
  - `Enter`: `focusedRow === 1`이면 `onStart(missions[selectedMissionIndex].id)`, `focusedRow === 2`이면 `onExit()`. `focusedRow === 0`일 때는 Enter에 별도 동작을 두지 않는다(미션은 좌/우로 이미 확정되어 있으므로).
- 마우스로도 동일하게 조작할 수 있도록, 게임 선택 행에 좌/우 화살표 버튼(◀ ▶)을 두어 클릭 시 `selectedMissionIndex`를 바꾸고, 시작/종료 버튼은 기존처럼 클릭 시 바로 실행되게 한다.
- 렌더링: 게임 선택 행은 "◀  Mission N  ▶" 형태로, 가운데 현재 선택된 미션 이름만 표시한다. 포커스가 이 행에 있을 때는 기존과 동일하게 강조 테두리/배경으로 표시한다.

## 5. 가정 및 향후 조정 여지

- 게임 선택 행에 포커스가 없을 때 좌/우 키를 누르면 아무 일도 일어나지 않는다(다른 행으로 이동하려면 위/아래 키를 써야 한다).
- 미션 전환은 순환(마지막에서 다음으로 가면 처음으로) 방식으로 하며, 이는 기존 위/아래 순환 방식과 일관성을 맞추기 위한 선택이다.
